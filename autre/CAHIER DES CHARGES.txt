CAHIER DES CHARGES — Projet PFA
Hunger-Talk : Application mobile intelligente de gestion nutritionnelle et alimentaire

1. Présentation générale du projet
Hunger-Talk est une application mobile intelligente qui aide l’utilisateur à mieux gérer son alimentation au quotidien.
L’application combine :
* une gestion automatique du stock alimentaire
* un système de recommandations de repas personnalisés
* un assistant virtuel basé sur l’IA
* une mise à jour dynamique du stock en fonction des plats cuisinés
L’objectif principal est de permettre à l’utilisateur de manger de manière équilibrée, d’éviter le gaspillage, et d’utiliser au mieux les aliments qu’il a déjà chez lui.

 2. Objectifs du projet
2.1 Objectif principal
Développer une application mobile intelligente permettant à l’utilisateur de :
* enregistrer ses aliments (manuellement ou via détection)
* consulter en temps réel son stock disponible
* exprimer ses envies ou ses objectifs nutritionnels
* obtenir des repas réalisables immédiatement selon son stock
* recevoir les recettes détaillées
* mettre à jour automatiquement son inventaire
2.2 Objectifs secondaires
* Simplifier la gestion des courses
* Réduire le gaspillage alimentaire
* Proposer un suivi nutritionnel quotidien
* Offrir une interface simple, intuitive et agréable
* Utiliser une IA moderne pour une interaction naturelle

 3. Public cible
* Étudiants et travailleurs souhaitant mieux gérer leur nutrition
* Personnes cherchant à optimiser leur planning alimentaire
* Familles voulant réduire le gaspillage
* Sportifs ayant des objectifs nutritionnels précis
* Toute personne souhaitant cuisiner selon ce qu’elle possède déjà

 4. Fonctionnalités principales
4.1 Gestion du stock alimentaire
* Ajouter un produit manuellement (nom, quantité, catégorie, date d’expiration)
* Ajouter un produit via photo (détection objet ? optionnelle)
* Possibilité de modifier ou supprimer un élément
* Affichage du stock par catégories
* Indicateurs visuels :
o produits presque finis
o produits expirant bientôt

4.2 Assistant IA (Chatbot)
Un chatbot intelligent basé sur un modèle de langage existant (LLaMA/Mistral) capable de :
* comprendre les messages naturels de l’utilisateur
* analyser les envies exprimées (ex : “j’ai envie d’un truc sucré”)
* tenir compte des objectifs nutritionnels (ex : “beaucoup de protéines aujourd’hui”)
* proposer une liste de repas réalisables avec le stock disponible
* afficher les valeurs nutritionnelles estimées

4.3 Système de recommandations
L’IA analyse :
* le stock actuel
* les préférences de l’utilisateur
* les restrictions alimentaires (halal, allergies, vegan…)
* l’objectif du jour (léger, protéiné, rapide, sucré…)
Et génère :
* 3 à 5 plats possibles
* sélection directe par l’utilisateur
* proposition d’alternatives si un ingrédient manque

4.4 Recettes
Pour chaque plat :
* Liste détaillée des ingrédients nécessaires
* Quantités à consommer
* Étapes de préparation
* Temps de cuisson
* Informations nutritionnelles (kcal, glucides, lipides, protéines)

4.5 Mise à jour automatique du stock
Lorsque l'utilisateur valide qu’il a préparé un plat :
* Les ingrédients utilisés sont automatiquement retirés du stock
* Les quantités sont recalculées
* Les produits arrivés à zéro sont supprimés ou notés comme “à racheter”

4.6 Notifications
* Avertissement d’expiration proche
* Propositions de recettes pour utiliser les produits avant qu’ils ne se périment
* Rappel nutritionnel quotidien
* Suggestions de courses

5.1 Application mobile
Technologies recommandées :
* Flutter (multiplateforme, rapide, gratuit)
ou
* React Native
5.2 Backend
Framework possible :
* Django (Python)
* Laravel (PHP)
* Node.js Express
Le backend assure :
* gestion utilisateurs
* gestion du stock
* communication avec le modèle IA
* stockage des recettes
5.3 Base de données
* PostgreSQL / Firebase / Supabase
Tables principales :
* users
* stock_items
* recipes
* nutrition_data
* preferences_users
5.4 Intelligence Artificielle
* Modèle existant (pas entraîné depuis zéro) :
o LLaMA 3.1 8B
o Mistral 7B
* Intégration via API interne
* Système RAG pour injecter stock + recettes à l’IA au moment de la conversation
5.5 Détection d’objet (optionnelle)
Utilisation possible :
* TensorFlow Lite
* YOLOv8-Nano
* ML Kit (Google)

 6. Interface utilisateur (UI/UX)
6.1 Ecrans principaux
* Page d’accueil (dashboard nutrition)
* Page de gestion du stock
* Scanner photo (si utilisé)
* Chat IA (page principale d’interaction)
* Page recette

6.2 Critères ergonomiques
* Design simple et moderne
* Navigation minimaliste
* Couleurs douces

 7. Sécurité & confidentialité
* Les données utilisateur doivent rester privées
* Stockage sécurisé (hash des mots de passe)
* Aucun transfert de données sensibles vers l’IA sans contrôle
* Possibilité de supprimer son compte et ses données

 8. Indicateurs de performance
* Temps de réponse de l’IA < 2 secondes
* Taux de détection correcte (si active)
* Pourcentage de stock correctement mis à jour
* Satisfaction utilisateur (questionnaire interne)
* Réduction du gaspillage (optionnel)

* gérer son stock facilement

* se voir proposer des plats réalisables immédiatement
* cuisiner selon ses objectifs
* suivre sa nutrition au quotidien
* réduire le gaspillage
* gagner du temps dans sa planification de repas
